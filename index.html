Biography

Photographer and designer based in Hammerfest, Hans Grecco Filipenco, has been honing is skills in geometric artwork for the past several years. Originally an italian cook from Marzamemi, Sicily - Hans blended his passion for savory dishes along with his keen eye for geometric abstractions, bringing about a perfect marriage of his two passions.

Hans' deep dive into technology has allowed him to introduce another element to his union and extend the range of his talent. By utilizing SVGs, Hans manifests his expressions in artwork through depths of movement and composition.

Documentation

  • CSS-grid and Flexbox-layout

    This project uses a combination of CSS-grid and Flexbox-layout to manage the responsive layout. The CSS-grid helps to align elements of the page in sequences, and plays an important role for the structure of this website. The site consists of three main elements, namely, the header, main-content, and footer. These elements belong to a layout defined under the #App-stylings in the CSS-file, and positions the header on top, followed by the main-content and the footer at the bottom.

    The use of Flexbox allows for responsive layout as the screen-size may differ from user to user. The content of this website is based of flex boxes consisting of columns with different breakpoints. Each column class can take a value from 1-12 points, and each row can hold up to 12 points before a shift in line, i.e. move the new column below the previous columns. As an example, we can have a row of 3 columns of 4 points each. If we were to add a new column to the row, this column would be placed below the three previous columns. Furthermore, columns can also have breakpoints (small, medium), which moves the column to a single row if the screen-size is lower than the breakpoint. To summarize, the use of Flexbox allows for an easy implementation of a fluid layout which changes along with the size of the users’ viewport.

  • HTML canvas and SVG in HTML

    The Canvas element is initialized in the HTML code, and the code for representing the art work is wrapped within a function. As a lot of the elements has the same characteristics, i.e. drawing of polygons from point to point – a helper function is used to reduce the amount of lines, and make the code more maintainable. The Canvas is redrawn on each frame and uses global variables to update the position, colours, rotation, etc. of the drawn elements to animate them. Furthermore, event listeners for the Canvas such as “mouseenter” and “mouseleave” are subscribed to in order to change the colors when the user hovers over the canvas.

    This site uses three SVGs whereas two of them are used as icons and is stored in files, and the third is the SVG-art itself and is produced inline/in-code. The SVGs are built and colored within the SVG-element, and key sub-elements are given classes which makes them easier to access from CSS. The animations and hover effects are handled by CSS. This is done by listening to state changes in the SVG (i.e. hover) and target the elements to animate/change by using the previously mentioned class.

  • Use of jQuery

    The jQuery library makes it easy to target HTML-elements and run functions with minimal lines of code compared to pure JavaScript. As this project has implemented scroll logic for the navigation bar and some sort of animation for each section, jQuery has been used to fill the gaps of CSS’ capabilities. The navigation bar changes state when the user scrolls down the page, and thus, never leaves the users’ viewport. This is done by listening to the scroll value of the mouse and adding a class to the navigation when a certain parameter is met. The same logic is implemented for each section, in order to trigger intro animations.

  • Cross-browser testing.

    As several browsers have their own interpretation of HTML, CSS, and JavaScript, it’s important to test that the site is user-friendly on all targeted browsers. The project has been developed with this in mind, and has been tested on three of the most used browsers, Firefox, Chrome and Safari. This was done by opening the .html file in each browser and manually run through a predetermined user test. Discrepancies was noted and solved until the site fulfilled the user test. As an addition, the site was also tested on different devices, i.e. iPhone and iPad, by using Chrome’s built-in developer tools.
